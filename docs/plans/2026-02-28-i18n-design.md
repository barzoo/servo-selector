# 多语言支持（i18n）设计方案

日期：2026-02-28

## 概述

为伺服选型工具添加英文界面支持，使用户可以在中文和英文之间切换。

## 技术选型

**使用 `next-intl` 库**
- 专为 Next.js App Router 设计
- 支持 TypeScript
- 支持静态导出（与 Vercel 部署兼容）
- 轻量且 API 简洁

## 架构设计

```
src/
├── i18n/
│   ├── config.ts          # i18n 配置（语言列表、默认语言）
│   ├── messages/
│   │   ├── zh.json        # 中文翻译
│   │   └── en.json        # 英文翻译
│   └── utils.ts           # 辅助函数（getMessages 等）
├── components/
│   └── LanguageSwitcher.tsx  # 语言切换组件
└── app/
    └── layout.tsx         # 集成 next-intl Provider
```

## 功能设计

### 语言切换器（LanguageSwitcher）
- 位置：页面右上角
- 样式：简洁的下拉选择或按钮组（中 / EN）
- 行为：
  - 点击切换语言
  - 保存选择到 LocalStorage
  - 更新 URL 参数（可选）

### 语言检测优先级
1. URL 参数 `?lang=en`（最高优先级）
2. LocalStorage 保存的偏好
3. 浏览器语言检测（如果匹配支持的语言）
4. 默认中文（最低优先级）

## 翻译内容范围

### 需要翻译的内容
- 所有 UI 标签（步骤名称、表单标签、按钮文字）
- 占位符提示文字
- 错误提示信息
- 页面元数据（title、description）
- 机制类型名称（滚珠丝杠 → Ball Screw 等）

### 不需要翻译的内容
- 电机型号（MC20-040-004-30-2）
- 技术参数名（rpm、N·m、kg）
- 计算结果数值

## 决策记录

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 切换方式 | 全局语言切换器 | 实现简单，不需要改变现有路由结构，与静态导出兼容 |
| 默认语言 | 中文 | 主要面向中国市场，保持现有用户体验 |
| 持久化方式 | LocalStorage + URL 参数 | 日常使用流畅，同时支持分享特定语言链接 |
