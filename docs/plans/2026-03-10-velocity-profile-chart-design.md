# 速度-时间曲线图设计文档

> **设计日期**: 2026-03-10
> **功能**: 在运动参数步骤提供简化的速度-时间曲线图，直观展示加减速过程

---

## 1. 需求背景

在"运动参数"步骤，用户需要选择运动曲线类型（梯形或 S 曲线）并输入相关参数。为了提供更直观的视觉反馈，需要在界面上添加一个简化的速度-时间曲线图，帮助用户理解所选曲线类型的形状特征。

---

## 2. 设计目标

- **简洁性**：极简设计，只展示曲线轮廓，不标注具体数值
- **实时性**：根据用户选择的曲线类型实时更新
- **一致性**：与现有 UI 风格保持一致

---

## 4. 技术方案

### 4.1 实现方式：SVG 路径绘制

选择 SVG 方案的原因：
- 无需额外依赖，纯 React + SVG
- 与现有代码风格一致（MotionStep 已使用 SVG 绘制曲线图标）
- 可轻松响应参数变化实时更新
- 文件体积小，性能优秀

### 4.2 曲线绘制逻辑

**梯形曲线 (TRAPEZOIDAL)**：
- 三段直线组成：斜线上升 → 水平线 → 斜线下降
- 使用 SVG `<path>` 的 `L` 命令绘制直线
- 时间比例：加速 25% / 匀速 50% / 减速 25%

**S 曲线 (S_CURVE)**：
- 使用贝塞尔曲线模拟平滑加减速
- 使用 SVG `<path>` 的 `C` 命令绘制三次贝塞尔曲线
- 起点和终点切线水平，中间过渡平滑

### 4.3 视觉样式

| 属性 | 值 |
|------|-----|
| 尺寸 | 宽度 100%，高度 120px |
| 曲线颜色 | `var(--primary-400)` |
| 曲线粗细 | 2px |
| 填充 | 半透明渐变 `var(--primary-500)/20` 到透明 |
| 坐标轴 | `var(--foreground-muted)`，1px |
| 动画 | 曲线切换时 200ms 过渡 |

## 7. 交互逻辑

1. 用户选择曲线类型（梯形/S曲线）
2. `profile` 状态更新
3. `VelocityProfileChart` 组件接收新的 `profile` 值
4. SVG 路径重新计算并渲染
5. 200ms 过渡动画完成曲线切换

---

## 8. 边界情况处理

- **初始状态**：默认显示梯形曲线
- **快速切换**：过渡动画期间防止闪烁
- **响应式**：SVG 使用 `viewBox` 确保在不同屏幕尺寸下正常显示
