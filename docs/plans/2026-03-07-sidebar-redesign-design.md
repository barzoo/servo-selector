# 侧边栏主导航重设计文档

**日期**: 2026-03-07
**功能**: 将侧边栏作为用户第一入口，整合项目信息、公共参数和轴配置
**设计**: 方案C - 混合方案

---

## 1. 概述

### 1.1 问题背景

当前UI存在以下问题：
- 项目设置按钮在顶部导航栏，难以发现
- 公共参数在第五步才显示，用户流程不合理
- "修改"链接触发事件但无监听
- "开始选型"按钮可能无反应

### 1.2 设计目标

- **侧边栏作为第一入口**：用户先确认项目和公共参数，再开始配置轴
- **清晰的信息架构**：项目信息 → 公共参数 → 轴列表
- **流畅的用户流程**：配置完成一个轴后自动返回侧边栏
- **易于发现的编辑入口**：在侧边栏直接编辑项目和公共参数

### 1.3 用户流程

```
用户进入应用
    ↓
侧边栏显示"新项目"，引导用户输入
    ↓
用户填写项目信息 + 确认公共参数
    ↓
用户点击"添加第一个轴"
    ↓
进入轴配置向导（机构→运动→工作条件→系统偏好→结果）
    ↓
完成一个轴后，自动返回侧边栏，显示已完成轴
    ↓
用户可以继续添加轴或导出报告
```

## 3. 组件设计

### 3.1 修改 AxisSidebar

添加以下功能：
- 项目信息展示区域（带编辑按钮）
- 公共参数展示区域（可折叠，带编辑按钮）
- 改进的轴列表项（带操作按钮）
- 空状态引导

### 3.2 修改 ProjectSettingsModal

- 支持两种模式：完整模式（项目信息+公共参数）和仅公共参数模式
- 添加打开/关闭的事件监听

### 3.3 修改主页面

- 添加空状态显示
- 移除顶部项目设置按钮
- 监听项目设置打开事件

### 3.4 修复 SystemConfigStep

- 修复"开始选型"按钮逻辑
- 简化公共参数提示（因为已在侧边栏展示）

## 4. 数据流

```
用户点击"编辑项目信息"
    ↓
打开 ProjectSettingsModal（项目信息标签页）
    ↓
用户修改并保存
    ↓
更新 project-store
    ↓
侧边栏自动刷新显示
```
