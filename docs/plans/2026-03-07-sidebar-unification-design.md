# 侧边栏统一化与轴命名功能设计文档

## 设计目标

1. 统一侧边栏元素交互模式，提升用户体验一致性
2. 添加轴命名功能，允许用户自定义轴名称
3. 修复主内容区指示器显示逻辑问题

## 当前问题分析

### 侧边栏结构不一致
- **项目信息区域**: 直接显示详情内容（名称、客户、销售）
- **公共参数区域**: 可展开折叠，显示所有参数详情
- **轴列表**: 简洁的Item模式，点击后在主区配置

### 主内容区指示器问题
编辑项目信息或公共参数时，主内容区仍显示"🛠️当前配置: 轴-X"，造成混淆。

## 设计方案

### 1. 统一侧边栏元素结构

将所有可配置项统一为相同的Item模式：

```
侧边栏
├── 📁 项目信息 (Item)
│   └── 仅显示项目名称，点击后在主区编辑
├── ⚙️ 公共参数 (Item)
│   └── 仅显示"公共参数"标题，点击后在主区编辑
├── 📊 轴配置 (区域标题)
│   ├── 🛠️ 轴-1 (Item) - 可点击、重命名、删除
│   ├── 🛠️ 轴-2 (Item)
│   └── ➕ 添加新轴 (按钮)
└── 📈 项目汇总 (区域)
    └── 统计信息和导出功能
```

**Item组件统一特征**:
- 左侧图标
- 名称/标题
- 右侧状态指示（编辑中/已完成）
- 点击后高亮显示
- 悬浮显示操作菜单（重命名、删除等）

### 2. 轴命名功能

#### 默认命名规则
- 添加第N个轴时，默认名称为 `轴-N`
- 名称在创建时自动生成，无需用户输入

#### 命名编辑
- 保留现有的AxisSidebarItem重命名功能
- 通过悬浮菜单访问"重命名"选项

#### Wizard内命名步骤
在Wizard流程的最后一页（第4步：系统配置）添加轴名称确认：
- 在系统配置表单底部添加"轴名称"输入字段
- 允许用户在此修改轴名称
- 作为配置完成的最后一步

### 3. 修复指示器显示逻辑

**当前代码** (page.tsx:221-229):
```tsx
{currentAxis && (
  <div className="mt-3 inline-flex items-center gap-2 px-3 py-1 bg-blue-100 text-blue-800 rounded-full text-sm">
    <span>🛠️</span>
    <span>当前配置: {currentAxis.name}</span>
    {currentAxis.status === 'CONFIGURING' && (
      <span className="text-xs">🔄</span>
    )}
  </div>
)}
```

**修复后逻辑**:
- 只在 `mainViewMode === 'wizard'` 时显示
- 只在有轴正在配置中 (`axis.status === 'CONFIGURING'`) 时显示

### 4. 侧边栏组件改造

#### 新的统一Item组件
创建 `SidebarItem` 通用组件，支持：
- 图标
- 标题/名称
- 编辑状态指示
- 激活状态样式
- 悬浮操作菜单

#### AxisSidebar改造
- 项目信息区域：使用SidebarItem，只显示项目名称
- 公共参数区域：使用SidebarItem，只显示"公共参数"
- 移除公共参数的展开/折叠功能
- 轴列表保持现有AxisSidebarItem

## 数据结构

无需修改现有数据类型，使用现有的：
- `Project` - 项目信息
- `AxisConfig` - 轴配置（已有name字段）
- `CommonParams` - 公共参数

## 交互流程

### 添加新轴
1. 用户点击"添加新轴"
2. 系统生成默认名称 `轴-${axes.length + 1}`
3. 切换到新轴的Wizard配置
4. 用户完成4步配置
5. 在第4步可修改轴名称
6. 完成后轴标记为COMPLETED

### 编辑项目信息
1. 用户点击侧边栏"项目信息"Item
2. 主区显示ProjectInfoEditStep
3. 侧边栏项目信息Item高亮显示"编辑中"
4. 主区不显示"当前配置"指示器

### 编辑公共参数
1. 用户点击侧边栏"公共参数"Item
2. 主区显示CommonParamsEditStep
3. 侧边栏公共参数Item高亮显示"编辑中"
4. 主区不显示"当前配置"指示器
