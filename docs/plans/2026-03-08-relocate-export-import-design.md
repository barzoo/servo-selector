# 迁移导入/导出功能到结果页面设计文档

## 概述

将项目数据导入/导出功能从顶部工具栏迁移到结果页面（ResultStep），与 PDF 导出按钮放在一起，形成统一的"项目输出"区域。同时从顶部工具栏移除工具菜单。

## 设计决策

### 布局位置
- **位置**: ResultStep 页面底部操作区域
- **布局**: 与 PDF 导出按钮并排，形成导出选项组
- **顺序**: 保存按钮 | 导出项目(JSON) | 导出 PDF | 导入项目

### 功能范围
- **导出项目**: 导出当前项目为 JSON 文件
- **导入项目**: 导入 JSON 文件覆盖当前项目
- **导出 PDF**: 保持现有功能
- **移除**: 顶部工具栏的 ProjectDataMenu 组件

### 交互设计
- 导出/导入按钮使用与 PDF 导出一致的样式
- 使用图标+文字的形式，清晰表达功能
- 导入/导出对话框保持现有设计

## 架构变更

### 文件变更
```
src/app/page.tsx                    # 移除 ProjectDataMenu 导入和使用
src/components/wizard/steps/ResultStep.tsx  # 添加导入/导出按钮
```

### 组件复用
- 复用现有的 `ExportDialog` 和 `ImportDialog` 组件
- 复用 `useProjectData` hook
- 无需修改对话框内部实现

## UI 设计

### 按钮布局
```
[返回编辑]                    [重新开始] [保存到篮子] [导出项目 📥] [导出 PDF 📄] [导入项目 📤]
```

### 按钮样式
- 导出项目: `btn btn-secondary` + 下载图标
- 导入项目: `btn btn-secondary` + 上传图标
- 导出 PDF: 保持现有样式

### 图标使用
- 导出项目: `Download` (Lucide)
- 导入项目: `Upload` (Lucide)
- 导出 PDF: `FileText` (Lucide)

## i18n 变更

### 新增翻译键
```json
{
  "result": {
    "exportProject": "导出项目",
    "importProject": "导入项目"
  }
}
```

### 移除翻译键
- 可选: 保留 `projectData` 命名空间以备将来使用，或完全移除

## 实现步骤

1. 修改 `ResultStep.tsx` 添加导入/导出按钮
2. 修改 `page.tsx` 移除顶部工具菜单
3. 更新 i18n 翻译文件
4. 测试验证

## 优势

1. **功能集中**: 所有导出相关功能在一个地方
2. **上下文清晰**: 用户在完成配置后自然能看到导出选项
3. **界面简化**: 移除顶部工具栏，减少视觉干扰
4. **符合心智模型**: 导出是结果页面的动作
